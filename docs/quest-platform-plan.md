# Quest Platform — Plan

Пешеходная квест-платформа, вдохновлённая квестом "По следам Фауста" (Вена, 2015).

---

## Механика

Игроки получают загадку → идут к месту → находят код → вводят в приложение → получают следующую загадку.

Коды можно оставлять на бумаге или через QR — оба варианта поддерживаются.

## База данных

```sql
quests
  id (uuid PK)
  title (jsonb)           -- {"ru": "...", "en": "...", "de": "..."}
  description (jsonb)
  city (text)
  is_published (bool)
  created_at

clues
  id (uuid PK)
  quest_id → quests
  order (int)
  content (jsonb)         -- текст загадки на всех языках
  hint (jsonb)            -- подсказка (показывается после N попыток)
  code (text)             -- верный ответ
  location_name (jsonb)
  lat, lng (float)        -- опционально, для карты
  media_url (text)        -- фото места / иллюстрация

teams
  id (uuid PK)
  quest_id → quests
  name (text)
  join_code (text)        -- короткий, 6 символов, напр. WLF-47
  created_at

sessions
  id (uuid PK)
  quest_id → quests
  team_id → teams?        -- null = соло
  device_id (text)
  nickname (text)         -- обязательный никнейм на старте
  recovery_code (text)    -- опционально, для смены устройства
  current_clue (int)
  lang (text)
  last_active_at
  started_at
  finished_at

attempt_log
  id (uuid PK)
  session_id → sessions
  clue_order (int)
  code_entered (text)
  is_correct (bool)
  created_at
```

## Мультиязычность

Два уровня:

1. **UI-строки** — react-i18next, JSON-файлы (`locales/ru.json`, `en.json`, `de.json`)
2. **Контент квеста** — JSONB в Supabase, каждое поле — объект с языками

Добавить новый язык = добавить ключ в JSONB и файл локализации.

Язык выбирается на стартовом экране (автоопределение из браузера + ручной переключатель).

## Режимы игры

### Соло

Открыть ссылку → ввести никнейм → играть. Сессия по device_id.

### Команда

- Лидер создаёт команду, вводит название → получает код `WLF-47`
- Делится кодом с участниками
- Остальные вводят код → подключаются к той же сессии
- Один прогресс на всех, Supabase Realtime синхронизирует мгновенно
- Код вводит лидер → все получают следующую загадку одновременно

---

## Идентификация игроков

- **Никнейм** обязателен перед стартом (соло) или название команды (команда)
- Без регистрации — сессия по device_id (uuid в localStorage)
- **Восстановление сессии** (опционально): короткий `recovery_code` выдаётся при старте, позволяет продолжить с другого устройства
- Для однодневного живого квеста восстановление необязательно

---

## Подсказки

После N неверных попыток показывается `hint` из таблицы `clues`.
N настраивается в админке на уровне квеста (например, 5 попыток).

---

## Брутфорс-защита

Rate limiting: 5 неверных попыток → 30 секунд блокировки.
Реализуется на клиенте + проверка `attempt_log` на сервере.

---

## Автоудаление сессий

| Тип             | TTL                         |
| --------------- | --------------------------- |
| Незавершённые   | 48 ч после `last_active_at` |
| Завершённые     | 7 дней                      |
| Ручное удаление | Всегда в админке            |

Реализуется через Supabase `pg_cron`.

## Админка (`/admin`)

Авторизация через Supabase Auth (email + пароль).
Row Level Security — анонимные пользователи не видят коды.

### Десктоп (редактор)

- CRUD квестов и загадок
- Drag-and-drop порядок загадок
- Редактирование текста на всех языках
- Загрузка медиа
- Тестовый режим (пройти квест без загрязнения статистики)
- Публикация / снятие с публикации

### Телефон (мониторинг во время квеста)

- Список активных игроков / команд
- Прогресс по загадкам
- Количество попыток (видно где застряли)
- Сброс на любой шаг
- Удаление сессии (= чистый старт для игрока)
- Пропуск сломанного этапа

## GDPR

Хранятся: device_id, никнейм, attempt_log.
Нужна плашка согласия при старте (мы в ЕС).
TTL на данные — см. автоудаление.

---

## QR vs ручной код

Оба варианта:

- QR-код на месте → сканируют камерой → код подставляется автоматически
- Ручной ввод → резервный вариант, если QR не сканируется

---

## PWA / офлайн

- Загадки кешируются после первой загрузки
- При потере связи показывается текущая загадка из кеша
- Код валидируется онлайн (нужен интернет для перехода к следующей загадке)

## Несколько квестов

Платформа поддерживает много квестов / городов.
Главная страница — список опубликованных квестов с фильтром по городу.
